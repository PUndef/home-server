# План перехода: замена Google + Obsidian и KeePass в Nextcloud

План по переносу файлов, документов, календаря, контактов, фото, задач, паролей (KeePass) и vault Obsidian в Nextcloud с работой на **Windows, Android и macOS**.

**Сервер:** Nextcloud на nextcloud-vm, [https://cloud-pundef.mooo.com](https://cloud-pundef.mooo.com) (см. [hardware-and-env.md](hardware-and-env.md), [nextcloud-setup.md](nextcloud-setup.md)).

Вся схема завязана на **одном** источнике правды — Nextcloud. Файлы, календарь, контакты, KeePass, Obsidian — без дополнительных слоёв синхронизации и сторонних «синк-сервисов»: клиенты подключаются к твоему серверу (веб, Desktop/Files, приложение, WebDAV, CalDAV/CardDAV).

---

## Как пользоваться

1. Выполняй шаги **по одному** — сначала сервер (приложения Nextcloud), потом клиенты по платформам.
2. После каждого шага делай **Проверить** и убедись, что всё работает на всех нужных устройствах.
3. Выполненные пункты переноси в таблицу **Сделано (история)** и помечай в заголовке шага статус **✅ сделано**.

---

## Общая схема замены


| Сервис Google / потребность | В Nextcloud                     | Клиенты (Windows / Android / macOS)                  |
| --------------------------- | ------------------------------- | ---------------------------------------------------- |
| Диск, файлы                 | Файлы                           | Desktop-клиент / приложение / Desktop-клиент         |
| Документы, таблицы          | ONLYOFFICE                      | Браузер / приложение (открытие в браузере) / браузер |
| Календарь                   | Приложение «Календарь» (CalDAV) | Любой CalDAV-клиент                                  |
| Контакты                    | Приложение «Контакты» (CardDAV) | Любой CardDAV-клиент                                 |
| Фото                        | Приложение «Фото» / Memories    | Загрузка через приложение или авто-загрузка          |
| Задачи, заметки *(опционально)* | Приложения «Задачи», «Заметки»  | Приложение или сторонние клиенты по CalDAV           |
| Пароли                      | Файл KeePass в папке Nextcloud  | KeePassXC / KeePassDX / Strongbox или KeePassXC      |
| Заметки Obsidian            | Папка vault в Nextcloud         | Obsidian + синхронизация папки                       |


Почта (Gmail) в этот план не входит: полноценная замена потребует отдельного почтового сервера; Nextcloud может быть только клиентом к существующему ящику (IMAP/SMTP).

## Подготовка на сервере (приложения Nextcloud)

Включи нужные приложения в **Настройки → Приложения**, чтобы дальше настраивать клиенты. Задачи и Заметки — **опционально**, если не нужны, можно не включать.

---

### Проверить, что уже установлено и включено

На nextcloud-vm (по SSH) выполни:

```bash
sudo -u www-data php /var/www/nextcloud/occ app:list
```

В выводе будут блоки **Enabled:** (включённые приложения) и **Disabled:** (установленные, но выключенные). Нужные для плана приложения имеют идентификаторы:

| ID приложения | Назначение        |
| -------------- | ------------------ |
| calendar       | Календарь (CalDAV) |
| contacts       | Контакты (CardDAV) |
| photos         | Фото (галерея)     |
| tasks          | Задачи *(опционально)* |
| notes          | Заметки *(опционально)* |

Только включённые приложения из блока **Enabled:** доступны пользователям. Если приложения нет в списке — его нужно установить из веб-интерфейса (Настройки → Приложения → Доступные).

**Только список нужных приложений:**

```bash
sudo -u www-data php /var/www/nextcloud/occ app:list | grep -E 'calendar|contacts|photos|tasks|notes'
```

Строки из блока **Enabled** означают, что приложение включено; если строка попала в **Disabled** — включи в веб-интерфейсе или командой `occ app:enable <id>`.

---

| Приложение | Назначение                      | Статус |
| ---------- | ------------------------------- | ------ |
| Календарь  | CalDAV для календарей           | ✅      |
| Контакты   | CardDAV для контактов           | ✅      |
| Фото       | Галерея по папкам               | ✅      |
| Задачи     | Задачи (поддержка CalDAV VTODO) *(опционально)* | ✅      |
| Заметки    | Текстовые заметки *(опционально)* | не установлен (опц.) |

### Шаг 0. Включение приложений Календарь, Контакты, Фото (+ опционально Задачи, Заметки) — ✅ сделано

**Цель:** на сервере доступны CalDAV, CardDAV и фото для клиентов на всех платформах. Задачи и Заметки — по желанию.

**Сделать:**

1. Проверить текущее состояние (команды выше). Что уже в **Enabled** — не трогать.
2. Войти в Nextcloud под администратором → **Настройки** (шестерёнка) → **Приложения**.
3. В разделе «Категории» или «Все» найти и **включить** (если выключены):
   - **Календарь** (calendar)
   - **Контакты** (contacts)
   - **Фото** (photos; может называться «Галерея» в зависимости от версии)
   - *(по желанию)* **Задачи** (tasks), **Заметки** (notes)
4. Включить через CLI (альтернатива веб-интерфейсу), на nextcloud-vm:
   ```bash
   sudo -u www-data php /var/www/nextcloud/occ app:enable calendar
   sudo -u www-data php /var/www/nextcloud/occ app:enable contacts
   sudo -u www-data php /var/www/nextcloud/occ app:enable photos
   # опционально:
   # sudo -u www-data php /var/www/nextcloud/occ app:enable tasks
   # sudo -u www-data php /var/www/nextcloud/occ app:enable notes
   ```
5. При необходимости установить приложение **Memories** (улучшенная галерея) из «Доступные» — по желанию. *На NC 33 Memories пока не поддерживается (макс. NC 32); использовать встроенное приложение «Фото» до выхода совместимой версии Memories.*

**Проверить:**

- Повторно выполнить `sudo -u www-data php /var/www/nextcloud/occ app:list` — calendar, contacts, photos в блоке **Enabled**.
- В главном меню (или в «+») видны пункты Календарь, Контакты, Фото (и при желании Задачи, Заметки).
- В **Настройки → Безопасность** (или «Подключение к аккаунту») отображаются ссылки для CalDAV и CardDAV.

## 1. Файлы и синхронизация (основа для Obsidian и KeePass)

На всех платформах нужен доступ к файлам Nextcloud: через приложение или через синхронизацию папки.

---

### Шаг 1.1. Nextcloud на Windows — ✅ сделано

**Цель:** синхронизация папки Nextcloud на ПК (в т.ч. для Obsidian и KeePass).

**Сделать:**

1. Скачать клиент для ПК с [nextcloud.com/install](https://nextcloud.com/install/#install-clients) — на Windows сейчас это **Nextcloud Files** (у Nextcloud несколько клиентов: Files, Desktop и др.; для синхронизации папки подойдёт Files или аналог с выбором папки).
2. Установить, запустить.
3. Указать адрес сервера: `https://cloud-pundef.mooo.com`, войти логином/паролем Nextcloud.
4. Выбрать папку для синхронизации на диске (например `C:\Users\<user>\Nextcloud` или `D:\Nextcloud`).
5. Выбрать, какие папки синхронизировать (или всё). Сохранить — дождаться первой синхронизации.
6. **По желанию:** в настройках клиента включить **только просмотр без скачивания** (файлы видны в проводнике, но не занимают место — скачиваются по открытию). И **не удалять локальные файлы** при освобождении места (если есть опция «Free space» / «Освободить место» — выставить «Never delete local files» / «Не удалять локальные файлы»), чтобы случайно не стереть то, что нужно локально (например KeePass/Obsidian).

**Проверить:**

- В выбранной папке появились файлы/папки из Nextcloud (или плейсхолдеры при режиме «только просмотр»).
- Изменение файла в папке отражается в веб-интерфейсе Nextcloud (обнови страницу).
- Иконка Nextcloud в трее показывает статус «Синхронизировано» без ошибок.

### Шаг 1.2. Nextcloud на Android — ✅ сделано

**Цель:** доступ к файлам Nextcloud с телефона/планшета (файлы, авто-загрузка фото при желании).

**Сделать:**

1. Установить приложение **Nextcloud** из Google Play.
2. Открыть приложение → **Войти** → ввести адрес `https://cloud-pundef.mooo.com`, логин и пароль.
3. При запросе разрешений — выдать доступ к файлам/хранилищу (для загрузки фото и сканов).
4. В настройках приложения при необходимости включить **Автозагрузка фото** (в выбранную папку в Nextcloud) и **синхронизацию** нужных папок (если есть опция).

**Проверить:**

- В приложении видны файлы и папки, как в веб-интерфейсе.
- Загрузка тестового файла с устройства в Nextcloud проходит успешно.
- Если включена автозагрузка — после съёмки фото оно появляется в указанной папке (после синхронизации).

---

#### Почему в выборе файлов виден Google Drive, а Nextcloud — нет

**Причина не в тебе и не в сервере.** Google Drive встроен в экосистему Android (приложение от Google), поэтому он **по умолчанию** регистрируется как источник в системном диалоге «Открыть файл» и показывается в списке. Nextcloud — стороннее приложение; его интеграция с Android Storage Access Framework (SAF) **неполная и багованная**: на многих устройствах и прошивках Nextcloud **вообще не появляется** в списке источников в файловом пикере, хотя приложение установлено и залогинено. Это известная проблема (см. [nextcloud/android#2203](https://github.com/nextcloud/android/issues/2203), [nextcloud/android#13716](https://github.com/nextcloud/android/issues/13716)). Итог: с «родным» облаком Google Android дружит из коробки, с Nextcloud — нет, поэтому для KeePass и других приложений остаётся один рабочий путь — открывать файл **из приложения Nextcloud** («Открыть в»).

**Что можно попробовать (без гарантии):** в системном приложении «Файлы» (Files / Менеджер файлов) зайти в меню → «Хранилища» / «Добавить хранилище» / «Облако» — посмотреть, есть ли пункт Nextcloud. На части устройств такого варианта **вообще нет**; где есть — после добавления Nextcloud может начать появляться в диалоге открытия. Если варианта нет — без вариантов, только «Открыть в» из приложения Nextcloud.

---

#### Важно: как KeePass и Obsidian видят файлы Nextcloud на Android

На Android приложение Nextcloud **не создаёт обычную «папку в системе»**, которую можно выбрать в настройках телефона. Синхронизированные файлы лежат во внутреннем хранилище приложения, и другие приложения получают к ним доступ **не через выбор папки**, а так:

- **KeePass (KeePassDX):** из самого KeePassDX выбрать облако Nextcloud в списке **не получается** (на практике так не работает). Рабочий способ один: открыть приложение **Nextcloud** → зайти в папку с базой → нажать на файл **.kdbx** → **«Открыть в»** / **«Open with»** → **KeePassDX**. Каждый раз так. После закрытия KeePassDX ссылка не живёт — снова открываешь Nextcloud и оттуда в KeePassDX.
- **Obsidian:** на десктопе vault в папке Nextcloud = мгновенная синхронизация. На Android приложение Nextcloud папку в систему не отдаёт; WebDAV (плагин Remotely-Save) **работает медленно** (по факту не «мгновенно»). Варианты на телефоне: либо редкий доступ через медленный WebDAV, либо пользоваться Obsidian в основном на ПК/Mac. Подробнее — [Шаг 7.2 (Obsidian на Android)](#шаг-72-obsidian-на-android--не-сделано).

## 3. Календарь и контакты (CalDAV / CardDAV)

Для всех ОС нужны клиенты с поддержкой CalDAV и CardDAV. Адреса возьми в Nextcloud: **Настройки → Безопасность** (или «Подключение к аккаунту») — там отображаются URL для CalDAV и CardDAV.

---

### Шаг 3.1. Календарь и контакты на Windows — ⬜ не сделано

**Цель:** календарь и контакты Nextcloud в приложении на ПК.

**Сделать:**

1. Установить клиент с CalDAV/CardDAV, например:
  - **Thunderbird** (календарь: расширение Lightning или TbSync; контакты: CardBook или TbSync), или
  - **Outlook** (при поддержке CalDAV/CardDAV через доп. плагины), или
  - отдельно: **DAVx⁵** не подходит (Android), на Windows — **Radicale** не нужен (сервер уже Nextcloud). Лучший вариант — Thunderbird + TbSync или Lightning + CardBook.
2. В Thunderbird: добавить учётную запись CalDAV (адрес из Nextcloud, логин/пароль) и CardDAV (адрес и учётные данные).
3. Адреса в Nextcloud обычно вида:
  - CalDAV: `https://cloud-pundef.mooo.com/remote.php/dav/calendars/<user>/<calendar>/`
  - CardDAV: `https://cloud-pundef.mooo.com/remote.php/dav/addressbooks/users/<user>/<addressbook>/`

Для простоты можно использовать один общий URL:

- CalDAV: `https://cloud-pundef.mooo.com/remote.php/dav`
- CardDAV: `https://cloud-pundef.mooo.com/remote.php/dav/addressbooks/users/<user>/contacts/` (путь уточни в настройках Nextcloud).

**Проверить:**

- В Thunderbird отображаются календари из Nextcloud, события синхронизируются.
- Контакты из Nextcloud видны в Thunderbird (или в CardBook). Создание события/контакта в клиенте появляется в Nextcloud.

---

### Шаг 3.2. Календарь и контакты на Android — ⬜ не сделано

**Цель:** календарь и контакты Nextcloud в стандартных приложениях «Календарь» и «Контакты».

**Сделать:**

1. Установить **DAVx⁵** (Google Play или F-Droid) — он связывает CalDAV/CardDAV с системным календарём и контактами.
2. Открыть DAVx⁵ → «Добавить учётную запись» → ввести базовый URL `https://cloud-pundef.mooo.com/remote.php/dav` (или только CalDAV/CardDAV URL из Nextcloud), логин и пароль Nextcloud.
3. Выбрать синхронизацию календарей и адресных книг.
4. В настройках Android разрешить приложению «Календарь» и «Контакты» использовать данные от DAVx⁵ (календари и контакты появятся в стандартных приложениях).

**Проверить:**

- В приложении «Календарь» видны календари Nextcloud, события совпадают с веб-интерфейсом.
- В «Контакты» отображаются контакты из Nextcloud. Изменение на телефоне синхронизируется в Nextcloud.

---

### Шаг 3.3. Календарь и контакты на macOS — ⬜ не сделано

**Цель:** календарь и контакты Nextcloud в «Календарь» и «Контакты» (Contacts).

**Сделать:**

1. **Календарь:** Системные настройки → Интернет-учётные записи (или Календарь → Добавить учётную запись) → **Другое** → CalDAV. Сервер: `cloud-pundef.mooo.com`, полный URL: `https://cloud-pundef.mooo.com/remote.php/dav`, имя пользователя и пароль — от Nextcloud.
2. **Контакты:** Контакты → Добавить учётную запись → Другое → CardDAV. Аналогично: сервер/URL Nextcloud CardDAV, логин и пароль.
3. Дождаться первой синхронизации.

**Проверить:**

- В приложении «Календарь» отображаются календари и события из Nextcloud.
- В «Контакты» видны контакты Nextcloud. Создание/редактирование на Mac отображается в веб-интерфейсе Nextcloud.

Фото хранятся в папках Nextcloud; просмотр — через приложение «Фото» или Memories. Загрузка — вручную или авто-загрузка из приложения Nextcloud на Android (см. Шаг 1.2).

### Шаг 4.1. Фото на всех платформах — ⬜ не сделано

## 5. Задачи и заметки *(опционально)*

Если не используешь — шаги 5.1 и 5.2 можно пропустить. Задачи доступны по CalDAV (VTODO), заметки — приложение Nextcloud или сторонние клиенты.

---

### Шаг 5.1. Задачи на Windows / Android / macOS *(опционально)* — ⬜ не сделано

**Цель:** задачи Nextcloud в выбранном клиенте на каждой платформе.

**Сделать:**

- **Windows:** Thunderbird + расширение для задач (например TbSync с типом «Задачи») или приложение **Nextcloud** (если есть встроенные задачи). Либо CalDAV-клиент с поддержкой задач (например OpenTasks на Android можно не использовать на Windows — тогда веб или приложение).
- **Android:** приложение Nextcloud — раздел «Задачи», либо **OpenTasks** + DAVx⁵ (DAVx⁵ синхронизирует и задачи с Nextcloud).
- **macOS:** приложение «Напоминания» (Reminders) можно связать с CalDAV; при добавлении учётной записи CalDAV в «Календарь» иногда предлагается и список задач. Либо использовать веб-интерфейс Nextcloud или клиент **Reminder** с поддержкой CalDAV.

**Проверить:**

- Задача, созданная в Nextcloud (веб), появляется в выбранном клиенте после синхронизации.
- Создание задачи в клиенте отображается в Nextcloud.

**Цель:** доступ к заметкам Nextcloud на всех устройствах.

**Сделать:**

- **Android:** в приложении Nextcloud открыть «Заметки».
- **macOS:** веб или приложение Nextcloud; при желании — сторонние клиенты с поддержкой Nextcloud Notes (если найдутся в App Store).

**Проверить:**

- Заметка, созданная в веб-интерфейсе Nextcloud, видна в приложении на телефоне и при открытии Nextcloud на другом ПК.

## 6. Пароли (KeePass) на всех платформах

Один файл `.kdbx` хранится в папке Nextcloud (например `Nextcloud/Passwords/Passwords.kdbx`). На каждом устройстве — свой клиент KeePass, открывающий этот файл из синхронизированной папки.

---

### Шаг 6.1. KeePass на Windows — ⬜ не сделано

**Цель:** открывать базу паролей из папки Nextcloud на ПК.

**Сделать:**

1. Установить **KeePassXC** ([keepassxc.org](https://keepassxc.org/)).
2. Создать новую базу (или перенести существующую) в папку, которая синхронизируется с Nextcloud (например `C:\Users\<user>\Nextcloud\Passwords\Passwords.kdbx`). Задать мастер-пароль.
3. Открывать этот файл в KeePassXC после синхронизации Nextcloud (Files/Desktop). При изменении базы сохранять — клиент Nextcloud синхронизирует файл.

**Проверить:**

- Файл `.kdbx` виден в папке Nextcloud на диске и в веб-интерфейсе.
- После изменения и сохранения в KeePassXC файл обновляется в Nextcloud (и на других устройствах после синхронизации).

---

### Шаг 6.2. KeePass на Android — ⬜ не сделано

**Цель:** открывать ту же базу паролей с телефона. На Android из KeePassDX выбрать Nextcloud в списке файлов **не выходит** — облако там нормально не открывается. Рабочий способ один: открыть базу **из приложения Nextcloud**.

**Сделать:**

1. Установить **KeePassDX** и приложение **Nextcloud**, войти в Nextcloud.
2. **Каждый раз так:** открыть приложение **Nextcloud** → перейти в папку, где лежит база (например `Passwords`) → нажать на файл **.kdbx** → в меню выбрать **«Открыть в»** / **«Open with»** → **KeePassDX** → ввести мастер-пароль.
3. После закрытия KeePassDX ссылка не сохраняется — следующий раз снова: открыть Nextcloud → папка → .kdbx → «Открыть в» → KeePassDX.
4. После изменений в KeePassDX нажать **Сохранить** — изменения уйдут в облако (пока сессия открыта из Nextcloud).

**Проверить:**

- База открывается: Nextcloud → файл → «Открыть в» → KeePassDX. После изменения на ПК и синхронизации при следующем открытии тем же способом изменения видны.

---

### Шаг 6.3. KeePass на macOS — ⬜ не сделано

**Цель:** открывать ту же базу паролей на Mac.

**Сделать:**

1. Установить **KeePassXC** для macOS ([keepassxc.org](https://keepassxc.org/)) или **Strongbox** (App Store).
2. Открыть файл из папки Nextcloud (например `~/Nextcloud/Passwords/Passwords.kdbx`). Ввести мастер-пароль.
3. Сохранять после изменений — Nextcloud Desktop синхронизирует файл.

**Проверить:**

- База открывается; изменения, сделанные на Windows или Android, после синхронизации видны на Mac (и наоборот).

## 7. Obsidian (vault в Nextcloud)

Vault Obsidian — обычная папка с файлами. **На Windows и macOS** её кладут в папку Nextcloud, которую синхронизирует клиент — синхронизация **мгновенная**. **На Android** без доп. решений Nextcloud папку не отдаёт, Remotely-Save (WebDAV) часто медленный. Реальные альтернативы для телефона: **FolderSync** (синхрон Nextcloud ↔ локальная папка, Obsidian открывает папку — быстрый отклик) и плагин **Folder Bridge** (WebDAV внутри vault). Конкретные шаги — в [Шаге 7.2](#шаг-72-obsidian-на-android--не-сделано).

---

### Шаг 7.1. Obsidian на Windows — ⬜ не сделано

**Цель:** vault Obsidian в папке Nextcloud, синхронизация на ПК.

**Сделать:**

1. Создать в Nextcloud папку, например `Obsidian` или `Vault` (через веб или в уже синхронизированной папке).
2. На ПК в папке Nextcloud (после синхронизации) создать или перенести папку vault (все `.md` и `.obsidian`).
3. В Obsidian: «Open folder as vault» → выбрать эту папку (например `C:\Users\<user>\Nextcloud\Obsidian\MyVault`).
4. Работать как обычно; после сохранения клиент Nextcloud (Files/Desktop) синхронизирует изменения.

**Проверить:**

- Изменение заметки в Obsidian сохраняется; в веб-интерфейсе Nextcloud в папке видны обновлённые файлы после синхронизации.

---

### Шаг 7.2. Obsidian на Android — ⬜ не сделано

**Цель:** доступ к vault с телефона с приемлемой скоростью. Remotely-Save (WebDAV) у тебя оказался медленным — ниже реальные альтернативы.

**Где уже всё ок:** Windows и macOS — vault в папке Nextcloud, клиент синхронизирует в фоне, мгновенно. Сервер тут кейс закрывает.

**На Android — варианты, которые стоит попробовать:**

---

#### Вариант А. FolderSync: Nextcloud → локальная папка, Obsidian открывает папку

Источник правды остаётся **твой Nextcloud**. Приложение **FolderSync** (Google Play) умеет двухстороннюю синхронизацию по WebDAV: папка в облаке ↔ папка на телефоне. Obsidian открывает **локальную** папку — всё читается/пишется с диска, синхронизация с сервером идёт в фоне по расписанию (например каждые 15 минут или чаще).

**Сделать:**

1. Установить **FolderSync** (есть бесплатная версия; в ней реклама и ограничения — при желании платная без рекламы).
2. В FolderSync добавить аккаунт **WebDAV**:
   - URL: `https://cloud-pundef.mooo.com/remote.php/dav/files/<твой_логин>/` (подставь логин Nextcloud)
   - Логин и пароль — от Nextcloud.
3. Создать **пару синхронизации**: удалённая папка — путь к vault в Nextcloud (например `/Obsidian/MyVault`), локальная папка — папка на телефоне (например «Документы / ObsidianVault»). Режим — **двухсторонняя** синхронизация. Расписание — по желанию (каждые 15 мин, 5 мин или при подключении к Wi‑Fi).
4. Запустить синхронизацию, дождаться первого обмена.
5. В **Obsidian** → **Open folder** → указать **локальную** папку (ту, что в п. 3). Дальше работаешь с ней как с обычным vault; FolderSync в фоне подтягивает/отправляет изменения на твой сервер.

**Минусы:** бесплатная версия с рекламой; у части пользователей бывают косяки при перемещении файлов между папками — при необходимости править структуру лучше на ПК. Плюс: быстрый доступ в Obsidian (локальный диск), синхр с Nextcloud в фоне.

---

#### Вариант Б. Плагин Folder Bridge в Obsidian (WebDAV «внутри» vault)

Плагин **Folder Bridge** ([GitHub](https://github.com/tescolopio/Obsidian_FolderBridge)) монтирует WebDAV (в т.ч. Nextcloud) как папку внутри vault — без отдельного приложения. На Android может вести себя лучше, чем Remotely-Save (другая схема доступа). Стоит попробовать: если скорость устроит — лишнего софта не нужно.

**Сделать:**

1. В Obsidian на телефоне: **Настройки** → **Сообщество** → включить плагины → **Browse** → найти **Folder Bridge** → установить и включить.
2. В настройках Folder Bridge добавить **WebDAV**: URL `https://cloud-pundef.mooo.com/remote.php/dav`, логин/пароль, путь к папке vault в облаке. Папка появится в боковой панели vault.
3. Работать с заметками через эту папку; сохранения уходят по WebDAV. Если скорость приемлема — остаёшься на этом варианте.

---

#### Вариант В. Remotely-Save (как есть)

Если готов к медленной синхронизации для редких правок на телефоне — оставить Remotely-Save. URL: `https://cloud-pundef.mooo.com/remote.php/dav`, логин/пароль, путь к vault.

---

#### Вариант Г. Obsidian только на ПК/Mac

Основная работа в vault на десктопе (папка Nextcloud = быстрая синхронизация). На телефоне не пользоваться Obsidian или заходить в файлы через веб Nextcloud для просмотра.

---

**Рекомендация:** для быстрого отклика на телефоне сначала попробовать **Вариант А (FolderSync)** — локальная папка даёт мгновенное открытие в Obsidian, синхронизация с твоим Nextcloud в фоне. Если не хочешь ставить FolderSync — попробовать **Вариант Б (Folder Bridge)** и оценить скорость.

**Проверить:**

- Вариант А: после синхронизации в FolderSync изменения из Nextcloud видны в локальной папке и в Obsidian; правки в Obsidian после следующей синхронизации появляются в Nextcloud.
- Вариант Б: папка из WebDAV открывается в Obsidian, сохранения уходят на сервер; скорость приемлема или нет — по ощущениям.

---

### Шаг 7.3. Obsidian на macOS — ⬜ не сделано

**Сделать:**

## Миграция данных из Google (файлы, календарь, контакты)

Ниже — как быстро и удобно перенести **файлы (Google Drive), календарь и контакты** в Nextcloud. Календарь и контакты — через экспорт/импорт вручную (это быстро, без плагинов). Файлы — вручную при малом объёме или через **rclone** при большом (облако-в-облако или через ПК).

---

### После Google Takeout: что делать дальше

Ты заказал выгрузку на [takeout.google.com](https://takeout.google.com). Когда архив(ы) готовы:

1. **Скачать до истечения срока** — ссылки на скачивание действуют **7 дней**. При большом объёме Google может разбить выгрузку на несколько архивов (например по 2 ГБ).
2. **Сохранить архивы** на диск и при необходимости **распаковать** (часто один общий .zip или несколько .zip — распаковать все в одну папку, например `Takeout`).

Типичная структура внутри Takeout (после распаковки):

| В Takeout бывает | Что это | Куда переносить в Nextcloud |
|------------------|---------|-----------------------------|
| **Drive** или **My Drive** | Папки и файлы из Google Drive | Файлы Nextcloud (см. ниже «Файлы из Takeout») |
| **Calendar** | Папка с .ics файлами (по календарям) | Календарь Nextcloud → Импорт календаря |
| **Contacts** | Файл контактов (All Contacts.vcf или .csv) | Контакты Nextcloud → Импорт |

Дальше — по типам данных.

---

#### Календарь из Takeout

1. В распакованном Takeout найти папку **Calendar**, внутри — файлы **.ics** (иногда один `Calendar.ics`, иногда несколько — по одному на календарь).
2. Открыть Nextcloud → **Календарь** → внизу слева **Настройки** (шестерёнка) → **Импорт календаря**.
3. Выбрать по очереди каждый .ics из папки Calendar. Указать календарь назначения (или создать новый). Повторить для всех .ics.

**Проверить:** в Nextcloud отображаются события; даты совпадают с Google.

---

#### Контакты из Takeout

1. В распакованном Takeout найти папку **Contacts**. Внутри обычно один файл: **All Contacts.vcf** (или контакты в .csv, в зависимости от выбора при создании Takeout).
2. Открыть Nextcloud → **Контакты** → внизу слева **Настройки** (шестерёнка) → **Импорт**.
3. Загрузить файл из папки Contacts (All Contacts.vcf или .csv). Выбрать адресную книгу или создать новую.

**Проверить:** контакты отображаются в Nextcloud; при проблемах попробовать отдельный экспорт только контактов из contacts.google.com в CSV и импорт этого CSV.

---

#### Файлы из Takeout (Drive)

В Takeout папка **Drive** (или **My Drive**) повторяет структуру твоего Google Drive: подпапки и файлы. Иногда большие файлы лежат как отдельные .zip — их нужно распаковать в ту же структуру или оставить как есть и открывать при необходимости.

**Вариант А — загрузка через папку Nextcloud (удобно на Windows/macOS):**

1. Установлен и подключён **Nextcloud Files** (или Desktop), синхронизируется папка на ПК (например `C:\Users\...\Nextcloud` или `~/Nextcloud`).
2. В этой папке создать, например, **GoogleDrive**.
3. Скопировать всё содержимое из распакованного Takeout → **Drive** (или **My Drive**) в **Nextcloud/GoogleDrive** (перетаскиванием или копированием в проводнике/Finder). Дождаться синхронизации.
4. Файлы появятся в Nextcloud в папке «GoogleDrive». Очень большие объёмы будут синхронизироваться долго.

**Вариант Б — загрузка через веб-интерфейс:**

1. В Nextcloud создать папку, например **GoogleDrive**.
2. Открыть её → **Загрузить** → перетащить папки/файлы из распакованного Takeout/Drive. Либо загружать по частям (браузер может не справляться с тысячами файлов за раз — тогда загружать по подпапкам).

**Вариант В — копирование с диска через rclone (если уже настроен Nextcloud как WebDAV):**

На ПК папка с распакованным Takeout, например `D:\Takeout\Drive`. Nextcloud настроен в rclone как remote `nextcloud`. Тогда:

```bash
rclone copy "D:\Takeout\Drive" nextcloud:GoogleDrive --progress --transfers 4
```

(Подставить свои пути и имя remote; на macOS/Linux — свой путь к папке.)

**Проверить:** в Nextcloud в выбранной папке видны те же папки и файлы; несколько файлов открыть и сравнить с оригиналом.

---

#### Массовое удаление .json после Takeout

В Takeout часто попадают служебные **.json** (метаданные Google). После копирования в Nextcloud их можно удалить массово.

**Вариант 1 — через веб Nextcloud (если .json немного и в одной-двух папках):**

Зайти в папку (например `GoogleDrive`), отсортировать по типу или имени, выделить .json (Shift+клик по диапазону или «Выделить всё», если в папке только они) → Удалить. Повторить для других папок, где есть .json.

**Вариант 2 — через rclone (если rclone уже настроен на Nextcloud):**

Удаляются только файлы с расширением .json в указанной папке. Сначала **пробный прогон** (ничего не удалит), потом без `--dry-run`.

```bash
# Подставь имя remote и путь к папке в Nextcloud (например GoogleDrive или корень)
rclone delete nextcloud:GoogleDrive --include "*.json" --dry-run
# Если список верный — то же без --dry-run:
rclone delete nextcloud:GoogleDrive --include "*.json"
```

Если .json лежат в корне файлов (не в подпапке), путь может быть пустым или `.`:

```bash
rclone delete nextcloud: --include "*.json" --dry-run
rclone delete nextcloud: --include "*.json"
```

**Вариант 3 — на сервере (если rclone не хочешь ставить):**

Удалять файлы лучше через WebDAV или веб, чтобы обновлялась база Nextcloud. Прямое удаление из каталога `nextcloud-data` на диске не рекомендуется (внутри структура может быть с хешами). Поэтому массово по расширению удобнее всего **rclone** (вариант 2). Если rclone недоступен — остаётся веб (вариант 1) по папкам.

**Проверить:** в Nextcloud в папке(ах) не осталось .json; нужные файлы на месте.

### Файлы (Google Drive → Nextcloud)

| Способ | Когда удобен | Примечание |
|--------|----------------|------------|
| **rclone** (CLI) | Много данных, нужна автоматизация, сохранение структуры папок | Бесплатно, работает на Windows/macOS/Linux и на сервере. Можно копировать Google Drive → Nextcloud WebDAV без полной выгрузки на диск. |
| **RcloneView** (GUI для rclone) | Хочется визуальный интерфейс, очереди, расписание | Платный GUI; под капотом тот же rclone. |
| **Вручную** (перетаскивание) | Мало файлов (несколько гигабайт), немного папок | Открыть drive.google.com и nextcloud в браузере (или Desktop) и перетащить. Медленно при большом объёме. |
| **Google Takeout** | Резервная копия «всё сразу» | takeout.google.com → выбрать Drive → скачать архив. Ссылки на скачивание действуют **7 дней**. Большие аккаунты (100+ ГБ) могут готовиться 24–48 ч. Потом архив нужно распаковать и загрузить в Nextcloud вручную или скриптом. |

**Рекомендация:** при объёме больше 10–20 ГБ или многих папках — настроить **rclone** один раз и перенести копированием Google Drive → Nextcloud (WebDAV). Так не нужно всё качать к себе и снова заливать.

---

#### Вариант: перенос файлов через rclone — ⬜ не сделано

**Цель:** перенести содержимое Google Drive в папку Nextcloud с сохранением структуры, без перетаскивания вручную.

**Сделать:**

1. Установить **rclone** на ПК или на nextcloud-vm: [rclone.org/downloads](https://rclone.org/downloads) (Windows/macOS/Linux).
2. Настроить удалённые (один раз):
   ```bash
   rclone config
   ```
   - **Remote 1:** тип `drive` — привязать свой Google-аккаунт (браузер или токен). Имя, например `gdrive`.
   - **Remote 2:** тип `webdav` — URL `https://cloud-pundef.mooo.com/remote.php/dav/files/<твой_логин>/`, WebDAV 2.0, логин/пароль Nextcloud. Имя, например `nextcloud`.
3. Проверка листинга:
   ```bash
   rclone lsd gdrive:
   rclone lsd nextcloud:
   ```
4. Копирование (пример — корень Drive в папку `GoogleDrive` в Nextcloud):
   ```bash
   rclone copy gdrive: nextcloud:GoogleDrive --progress --transfers 4
   ```
   Вариант с копированием только изменённого (повторный запуск):
   ```bash
   rclone copy gdrive: nextcloud:GoogleDrive --progress --transfers 4 --update
   ```
   Не использовать `rclone sync` без понимания: sync удаляет в назначении то, чего нет в источнике.

**Проверить:**

- В Nextcloud в папке `GoogleDrive` (или выбранной) видны те же папки и файлы, что в Google Drive. Открыть несколько файлов — содержимое совпадает.

### Календарь (Google Calendar → Nextcloud)

Плагин не нужен: экспорт из Google в формате .ics и импорт в Nextcloud. Занимает несколько минут.

---

#### Шаг миграции: экспорт календаря из Google и импорт в Nextcloud — ⬜ не сделано

**Цель:** все события из Google Calendar оказались в Nextcloud Календарь.

**Сделать:**

1. **Экспорт из Google:**  
   Открыть [Google Calendar](https://calendar.google.com) → шестерёнка → **Настройки** → в левом меню **Импорт и экспорт** → **Экспорт**. Скачается .zip с одним или несколькими файлами .ics (по одному на календарь).
2. Распаковать .zip на компьютер.
3. **Импорт в Nextcloud:**  
   Открыть Nextcloud → приложение **Календарь** → в левом нижнем углу **Настройки** (шестерёнка) → **Импорт календаря** → выбрать .ics файл (или по одному для каждого календаря). Указать, в какой календарь Nextcloud импортировать (или создать новый).
4. Повторить импорт для каждого .ics, если календарей несколько.

**Проверить:**

- В Nextcloud в разделе «Календарь» отображаются перенесённые события и даты. Сравнить несколько событий с Google Calendar — совпадают.

**Если импорт ругается на отдельные события:** в экспорте Google иногда встречаются битые поля (например, дата создания «0001»). Тогда импортировать по одному календарю или открыть .ics в текстовом редакторе и удалить/исправить строки с годом 0001, затем импорт повторить.

### Контакты (Google Contacts → Nextcloud)

Тоже без плагина: экспорт из Google, импорт в приложение «Контакты» Nextcloud.

---

#### Шаг миграции: экспорт контактов из Google и импорт в Nextcloud — ⬜ не сделано

**Цель:** контакты из Google Контактов перенесены в Nextcloud Контакты.

**Сделать:**

1. **Экспорт из Google:**  
   Открыть [contacts.google.com](https://contacts.google.com) → слева **Экспорт** (или выбрать контакты → три точки → Экспорт). Выбрать **Google CSV** (предпочтительно для совместимости) или **vCard (.vcf)**. Скачать файл.
2. **Импорт в Nextcloud:**  
   Открыть Nextcloud → приложение **Контакты** → внизу слева **Настройки** (шестерёнка) → **Импорт** → загрузить сохранённый файл (CSV или .vcf). Выбрать адресную книгу или создать новую.
3. Дождаться окончания импорта.

**Проверить:**

- В Nextcloud «Контакты» отображаются перенесённые контакты. Открыть несколько — имена, телефоны, почта на месте.

**Важно:** при экспорте vCard из Google у части контактов возможны «тихие» ошибки (не все поля или не все контакты попадут). Если что-то пропало — попробовать экспорт в **CSV** и импорт CSV в Nextcloud (поддержка зависит от версии) или разбить экспорт на несколько меньших .vcf и импортировать по частям.

---

### Краткая последовательность миграции (только файлы / календарь / контакты)

1. **Календарь** — экспорт .ics из Google → импорт в Nextcloud Календарь (2–5 минут).  
2. **Контакты** — экспорт CSV или .vcf из Google → импорт в Nextcloud Контакты (2–5 минут).  
3. **Файлы** — при малом объёме: выгрузить нужные папки с drive.google.com и загрузить в Nextcloud (веб или папка Desktop). При большом объёме: настроить rclone и перенести одной командой (см. выше).

После переноса подключи клиенты (DAVx⁵, Thunderbird, системные Календарь/Контакты на macOS) к Nextcloud — данные уже там.
